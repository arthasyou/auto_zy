

【任务内容】

背景：
某大语言模型已学习大量中医古籍、现代医案及中医理论，但在实际应用中存在混乱、矛盾的问题。现有一套【待生成语料的中医资料】，用于生成新的微调语料，以提升模型在中医问答领域的准确性与一致性。

目标：
基于提供的文献资料，抽取深层次且不重复的中医知识，生成标准 JSON 格式的微调问答语料，包含四个字段：id / kind/ input/ instruct / output。

【执行要求】

1. 知识来源限定：

* 仅允许使用【原始医案】与【知识点】文件中的内容，不得引入任何外部中医理论或常识；
* 如资料与常识冲突，以资料为准。

2. 内容深度与多样性：

* 充分挖掘中医理论深层思想，覆盖症状、病机、方药、机理等逻辑链；
* 包括"从症状到选用药材"的反向推演。

3. 问答形式要求：

* 形式多样，符合真实中医诊疗交流风格；
* 语言专业、简明、准确，便于理解。

4. 结构与格式要求（必须遵守）：

* 回复内容必须是一个 JSON 代码块，使用 `json 开头，` 结尾；
* JSON 字段包括：id、kind、input、instruct、output，缺一不可；
* kind 字段根据【语料分类表】选择，格式为 "一级分类-二级分类"；
* 不得输出任何正文、解释、说明、问候、总结等额外内容，只能输出符合规范的 JSON。

5. input / instruct / output 构建标准：

* input：

  * 从【原始医案】文件中找到最匹配 instruct 和 output 聚焦信息的病案。
  * 根据【知识点】文件中的背景资料，抽取与 instruct 和 output 对应的知识结构。
  * 二者在 input 中分别列出。
  * 【重要】仅展示患者的信息和必要的聚焦内容，其他部分简化，因此 input 总篇幅控制在几百字内。

* instruct：

  * 以提问或指令形式设计，灵活多样；

* output：

  * 基于【知识点】文件中的内容进行详细、专业、准确的回答，不可简略概括。

6. 语料分类表（kind字段使用，确保每个子分类都有涉及，且数量上尽可能平均）：

* 病机分析：1.总病机分析；2.症状到病机分析；3.病机定位分析；4.病因分析；8.治疗机理分析；
* 治法选择：5.治疗方法选择原理；6.治疗原则确定；7.治疗路径选择原理；11.病症到药物配置分析；
* 方药解析：10.单味与组方治机理分析；12.前后处方关联；14.君药选择原理；15.臣佐使药选择原理；16.药材交互作用；17.煎药规则；18.服药规则；
* 反馈评估：9.治疗后健康转化分析；13.处方效果分析；19.基于症状改善分析药效及机理；20.基于症状改善分析病机转化。

7. 每次生成要求：

* 前三次回复生成计划，计划中每一条都是用简短的话描述一条语料要问答的内容，这三次计划各包含100条语料的计划（包含编号）；从第四次回复开始，就按照计划逐条生成语料，且确保每条语料都是逐条来自计划中；
* 从第四次开始，每次仅生成1条语料；
* 每条语料是独立的，自洽的，基于已有中医资料内容进行提取，不得引用其他语料；
* 后续我发送“1”时，你就按照上述要求先生成三次计划，从第四次开始回复语料，每次回复语料都是从上次编号顺延生成新的语料。

8. 输出格式示例：

```json
{
  "id": "1",
  "kind": "病机分析-病因分析",
  "input": "(原始医案:) (中医资料)",
  "instruct": "请分析上述患者的病因形成机制。",
  "output": "基于原文提取的专业中医病因分析内容"
}
```

9. 连续生成能力：

* 没有新的可挖掘信息来继续生成符合要求的语料时，则变化形式与内容，灵活变化形成新的语料，以覆盖用户可能根据要求与资料问出的各种问题；
* 因此你的任务应当可以持续产生新的语料。

10. 药材重量单位更改： 

对于资料中出现的药材，其后可能跟随数字是其重量，无论资料原文中重量单位是哪个，该重量单位都应在数字大小不变的前提下被更改为“小两”，用于你后续语料的生成
